类的实例化、字段、方法

类的简单示例

在解决方案上右键选择添加即可创建一个类。

类的语法：

在 Main 函数中可以使用 Person person = new Person(); 根据类创建一个对象，这个过程叫做类的实例化。

类和对象：类是一个模板，确定了一个对象应该有的属性和方法。

要想类或者类中的成员可以被外界直接访问到需要对其使用 public 修饰符，代表公开的，如果不加修饰符，则默认为 private 修饰符，代表私有的，仅类的内部可以访问。

public string _name; 这样就定义了类中的一个字段，它的作用是存储数据。

这里类中的 SayHello 方法，没有使用 static 修饰符，因此称为非静态方法，使用 static 修饰符修饰的方法称为静态方法。

this 关键字代表当前类对象，一般在类中方法使用到该类的字段时，应加上使用 this 关键字调用，如 this._name;，但有时不加也没有明显的错误。

但是如果该类的 SayHello 函数中，定义了相同的 _name 字段，这时我们想调用类中的 _name 字段时，就需要使用 this 关键字，否则编译器并不能准确知道我们到底想使用哪一个 _name 。

属性的介绍

应该给每个字段都配备一个属性，属性的作用是保护字段，属性的书写方式如下所示

字段的命名规范一般是下划线加小写单词，与其对应的属性一般是该单词的首字母大写形式。

属性中分别有对该字段的 get 和 set 方法，分别用于取值和赋值，这样就可以将字段的修饰符改为 private ，仅类的内部可以访问，而在类之外通过与之对应的属性进行访问。

set 方法没有返回值，但是有一个与该字段一样类型的变量 value ，用于给字段赋值。
get 方法没有参数，但是有一个与该字段一样类型的返回值，用于把字段返回给外界。

字段是存储数据的，所以通常来说，字段应该是私有的，而外界访问应该使用属性来访问字段。

在类的外部通过属性来访问字段

这里虽然访问的是属性，但是属性内部的 get 和 set 方法最终会作用到字段上。

对象创建好后，依次给对象的每个属性赋值，这个过程叫做对象的初始化。

属性的作用

这就起到了属性的保护字段的作用，在属性的 set 方法中，对传入的值进行非法判断，保证存储在字段中的数据是合法的。这样在调用时，无法直接访问字段，只能通过属性，即使出现了类似 person.Gender = '中' 的非法情况，通过属性中的 set 方法，也可以将值改回默认值并将值赋给字段。同样在 get 方法中也可以做判断，保证调用者拿到的数据是合法的。除了合法性的判断，也可以对数据做处理，例如在 Name 属性中对传入的 value 进行去头尾空格的处理。

通常我们把字段设为私有，属性设为公开，涉及到访问字段的情况，就使用属性。

属性分为三种

  • 可读写属性：拥有 get 和 set 方法
  • 可写属性：只拥有 set 方法
  • 可读属性：只拥有 get 方法

自动属性

这种属性叫做自动属性，这样就需要写字段了，编译器会根据属性自动生成字段。因为我们不会直接去访问字段，所以也无需关心编译器生成的字段的名称，只要能存储数据就是可以的。但是这样的写法就不能在属性里对字段的值进行限制，只能在构造函数中限制。

构造函数

构造函数：对对象进行初始化。

之前对对象的初始化，需要挨个对每个字段去赋值，十分的麻烦，构造函数可以简单的完成这个工作。

构造函数其实就是一个函数，只不过是一个特殊的函数。

构造函数的语法

构造函数的特殊性：

  • 没有返回值，也不用写 void 。
  • 函数名称必须和类名相同。

当使用 new 关键字创建对象的时候，会调用构造函数。

new 的作用

  1. 在内存中开辟空间
  2. 在开辟的空间中创建对象
  3. 调用对象的构造函数

所以，构造函数的访问修饰符必须为共有，否则无法创建对象

如果我们没有声明构造函数，那么会默认有一个空参数的构造函数

当我们自定义了一个构造函数的时候，不论定义的构造函数有参无参，都会将原来的构造函数替换掉。

一个类可以有多个不同参数的构造函数，也就是构造函数的重载。

我们除了在属性的 get 和 set 方法中保护字段，也可以在构造函数中对字段进行保护。

可以看出，通过自定义的构造函数来初始化对象更加简单。

值类型和引用类型

值类型存储在栈当中，引用类型存储在堆当中

常见的值类型：int double char bool struct enum

常见的引用类型：string 数组 自定义类

值类型的值存储在内存中的栈中。

引用类型的值存储在内存中的栈中，而堆中存储的是栈的地址，使其指向栈的指定位置。

静态类和非静态类的区别

有 static 修饰的成员就是静态成员，没有被 static 修饰的就是非静态成员。

  1. 在非静态类中，是否允许出现静态成员？可以。
  2. 在非静态函数中，能不能够访问到静态成员？可以，也可以访问非静态成员。
  3. 在静态类中，是否允许出现非静态成员？不可以，在静态函数中只允许出现静态成员。
  4. 在静态函数中，能不能够访问到非静态成员？不可以，静态函数只能访问非静态成员。

在调用实例成员的时候，需要创建对象后再调用，例如 Perosn p = new Peson(); p.Name = "张三"。

在调用静态成员的时候，只需要使用类型+点(.)+成员名的方式就可以，例如 Console.WriteLine();

静态类是不允许创建对象的。

编写工具类的时候，通常使用静态类。静态类的好处就是资源共享。不需要每次使用都要 new 一个对象。静态类存储在内存中除堆和栈之外的静态存储区域。所以编写的类常常需要用到的工具类，则可以考虑使用静态类。

但是静态类应该越少越好，因为不管你使用或不试用，静态类都会占用内存，而且静态类只有在程序结束的时候，才会释放资源，总结来说还是占用资源。

不同项目间的类的引用

一个解决方案中可以含有多个项目，所以涉及到跨项目引用类。

首先先添加引用，然后再引用命名空间。

假设有一个项目 Project1 和 Project2 ，在 2 中使用 1 中的类。首先在 2 项目中，引用项目 1 ，然后在[要使用 1 中的类]的类中，引用名称空间 using Project1。

但是需要保证被引用的类的访问修饰符是 public。

修饰类的访问修饰符有两个 public 和 internal 。

internal 的访问权限仅限于当前程序集内部访问。

类的默认访问修饰符是 internal 。

字符串到字符数组的转换以及高效的 StringBuilder

C# 中 String 类型有一个重要特性：不可变性，字符串一旦被声明，在堆中存储的值就无法被改变，重新赋值时，只能在堆中开辟一块新的空间并指向它，但是原来的空间还依然存在。

因为 String 类型是引用类型，所以每次对 String 变量重新赋值，都会重新开辟一块空间，如果频繁的改变字符串变量值，就需要频繁的开辟空间，所以会产生很多内存垃圾。

使用 StringBuilder 则不会有这种问题，它并不会频繁的开辟空间。
