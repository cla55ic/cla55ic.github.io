FileStream

File 类可以方便的操作小文件，但是如果是操作大文件，速度就很慢。原因是因为 File 类采用的方式是一次性读写，会将文件一次性读取到内存当中，所以对内存的消耗很大，速度也比较慢。对于大文件的操作，一般使用 FileStream 文件流。

FileStream 用来操作字节、StreamWriter 和 StreamReader 用来操作字符。

FileStream 可以用来操作所有文件，StreamWriter 和 StreamReader 可操作文本文件。

装箱和拆箱

装箱：将值类型转换为引用类型，且两类型有继承关系
拆箱：将引用类型转换为值类型，且两类型有继承关系

两者没有继承关系的，不属于装拆箱操作。（即使有关系，有时也不是装拆箱子操作???）

装箱和拆箱的操作，转换类型要统一

在程序中，要尽量少的进行这种操作，因为装箱和拆箱都涉及到类型转换，转换量大之后会明显的影响程序运行效率。

泛型集合

两个泛型集合：List 和 Dictionary ，分别对应集合 ArrayList 和 Hashtable 。泛型集合相对于数组的好处就是长度不固定，相对于集合的好处就是不涉及到装箱和拆箱的操作，因为泛型集合是长度固定的，这也是泛型集合的缺点。

泛型集合的添加单个元素、添加集合元素、删除元素、清空数组和排序等等的方式和集合一样。

泛型集合在名称空间 System.Collections.Generic 下。

抽象类的要点

  1. 使用 abstract 修饰抽象类和抽象方法。仍然使用 override 来表示重写父类的方法。
  2. 被标记为 abstract 的抽象成员，不能有方法体。
  3. 抽象成员只能在抽象类中。
  4. 抽象类不能被实例化。但是抽象类是有构造函数的
  5. 继承于抽象类的子类必须重写父类中的抽象成员，除非子类也是抽象类。而子类继承的普通父类中的虚方法，子类可以选择性的重写。
  6. 抽象类的成员不能被 private 修饰，因为这与第 5 条矛盾。
  7. 在抽象类中可以有实例成员，作用是被子类所继承，同样的，抽象方法存在的作用就是被子类重写。
  8. 在子类中重写抽象父类的中的抽象方法时，需要保证函数的签名和返回值一致。

抽象方法与虚方法的的区别如下表

虚方法 抽象方法
用 virtual 修饰 用 abstract 修饰
有方法体 没有方法体
可以被子类重写 必须被子类重写
除了密封类都可以写 只能写在抽象类中
