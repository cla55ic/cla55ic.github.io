C# 学习笔记-继承、new、this、里氏转换、File类

发表于 2017-05-24

继承

在编写类的过程中，有可能出现多个类具有相同的属性或方法，例如学生类和老师类，都具有姓名、年龄和性别属性，都具有打招呼的方法，同时他们也具有自己独有的属性和方法，例如学生类具有学习方法，老师类具有教学方法。这时我们可以进一步抽象出人类。使人类具有姓名、年龄和性别属性，具有打招呼的方法，然后我们将学生类和老师类继承人类，就可以继承到人类所具有的成员，而只需要在自己的类当中编写自己所独有的成员就可以了。

可以看出，这样我们也不必去编写重复的代码，所以继承可以有效的解决代码冗余的问题。

我们称学生类继承了人类，人类为父类，学生类为子类。或者说人类派生了学生类，人类为基类，学生类为派生类。

子类继承了父类的属性和方法。但是因为字段一般是私有的，所以一般情况下子类不能继承父类的字段。除非字段不是私有的。子类也没有继承父类的构造函数。

子类虽然没有继承父类的构造函数，但是子类的构造函数会调用父类的默认无参构造函数。原因是因为子类继承了父类，所以需要使用父类中的成员，想要使用父类中的成员，就必须有一个父类对象，子类构造函数调用父类构造函数的目的就是使用父类成员。

假如父类 Person 类中只有一个有参构造函数

这时子类就无法调用到父类的无参构造函数，因为该函数已经被替换掉，这时就会报错。

此时有两个解决办法，一是给父类加一个无参构造函数，第二是使子类的构造函数调用父类的有参构造函数，我们通常使用第二种方法，示例如下

继承的特点

  1. 单根性，一个类只能有一个父类
  2. 传递性，子类拥有父类的成员，也拥有父类的父类的成员，以此类推。成员之间通过继承是向下传递的。

我们可以通过右击项目，查看类图，可以方便的看出各个类之间的关系。

在 C# 中，所有的类都直接或间接的继承了 Object 类，如果我们编写的类没有指明父类，会默认继承 Object 类。所以 Object 类是一切类型的父类。

new 的用法

new 可以创建对象，它还有另外一个作用。

当子类中有与父类相同的成员，例如名称相同的属性，名称和参数相同的方法时，会覆盖掉父类中的成员。这时可以在子类中的那个成员上加上 new 关键字已确认覆盖。

所以除了创建对象外，new 关键字还可以用来确认隐藏或者说覆盖从父类中继承的成员。

this 的用法

this 代表当前类的对象。另外一个作用是显式的调用自己的构造函数。

Student 类有两个构造函数

两个构造函数中含有部分相同的赋值代码，属于冗余代码。这时就可以使用 this 关键字，使其在构造函数中显式的调用另一个构造函数。

只能从少参数的构造函数调用多参数的构造函数，由于少参的构造函数不含有部分多参构造函数的参数，所以这里需要给该参数一个默认值，例如少参数的构造函数不含有多参数构造函数中的 ID 参数，所以这里传入一个 int 类型变量的默认值 0 。

里氏转换以及 as 和 is 的用法

里氏转换

  1. 子类对象可以赋值给父类(如果有一个地方需要一个父类作为参数，可以传入一个子类代替)。
  2. 如果这个父类当中装的是子类对象，那么可以将这个父类强转为子类对象。

String.Join(string separator, params object[] values)

该方法需要一个 可选的 object 数组，根据里氏转换，我们可以传入一个 object 的子类 string 类型的数组。

因为该方法需要一个 可变参数数组，意味着既可以放入数组，也可以放入单个的元素，又因为该数组是 object 类型数组，object 是一切类型的父类，所以可以放任意类型、任意个数的单个元素

假如父类 Person 中装的是 Student 对象，我们尝试转换为 Teacher 对象，(虽然父类中装的是子类对象，可以强制转换为子类，但也应该是对应的子类)，就会抛出异常。为了避免这种情况，应在转换前进行检查。

is：类型转换，如果转换成功，返回 true，失败，返回 false 。
as：类型转换，如果转换成功，返回相应对象，失败，返回 null 。
