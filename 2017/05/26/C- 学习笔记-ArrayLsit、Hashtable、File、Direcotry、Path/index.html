ArrayList

数组长度固定，而集合的特点就是长度不固定，会随着集合内的元素个数而成 2 的 n(n>=2) 次方倍增。ArrayList 就是集合的一种类型。

ArrayList 类不是静态类，所以想要使用该集合，要先实例化一个对象。ArrayList 类的 Add 方法需要一个 object 类型的值，根据里氏转换，我们可以传入任意类型的值。ArrayList 需要传入 object 类型是为了通用性。

向数组一样，集合也可以使用 for 循环通过下标来遍历

使用 AddRange 添加集合

因为 ArrayList 传入的 object 类型的参数，所以插入数据十分方便，但是如果我们插入了多个类型的值，在取数的时候就会变得困难，因为我们取出的也是 object 类型的值。

Count 表示集合当前包含的元素个数
Capcity 表示集合当前可存储的元素个数

可以看出一旦集合中实际包含的元素个数将要超过可以存储的元素个数时，集合的长度会成 2 的 n(n>=2) 次方倍增。

删除元素也会以相似的规则缩减集合的可包含元素的长度。

File

除了之前提到的 File 类的方法之外，File 类还可以读写文件。

读取文件前应注意以下几个点。

编码：将字符串以什么样的形式保存为二进制，这个就是编码，UTF-8, ASCII, Unicode 等。

读取文件产生乱码的原因：保存时的编码格式和读取时的编码格式不一致，就可能产生乱码。

文本文件：简单的解释，能被文本编辑器打开，且能正确识别内容的即是文本文件。

File 类中有三个方法可以一次读取文件中的所有内容。

File.ReadAllLines() 该方法按行读取文件。需要一个路径字符串作为参数，另外有一个重载可以指定其读取时采用的编码方式，根据行数返回一个字符串类型的数组。

有时我们只是单纯的想把文本文件的内容读取出来，而不需要按行读取成数组，则可以使用 File.ReadAllLines()，它和 File.ReadAllLines()，需要的参数一样，且也有指定编码类型的重载，而它返回的是字符串类型的值。

除了读取文本文件，还可以使用 File.ReadAllBytes() 来读取文件内容的字节形式。需要一个路径字符串作为参数，没有指定编码的重载，因为是是按字节读取，所以不需要编码。返回一个字节数组。

使用 Encoding.UTF8.GetString() 来将字节数组按 UTF-8 编码转换为文本。

Path

Path 类对路径字符串进行操作，对文件并没有影响。

绝对路径指从盘符开始的路径，例如 “E:\abc\test.txt” 。
相对路径指的是从当前路径开始的路径，这里当前路径指的就是可执行文件所在的路径，将 test.txt 放在当前项目的 Debug 文件夹下的 abc 文件夹下(可执行文件所在的文件夹)，那么它的相对路径就是 “abc/test.txt” 。
