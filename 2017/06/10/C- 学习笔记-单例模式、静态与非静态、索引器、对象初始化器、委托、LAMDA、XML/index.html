单例模式

单例模式的作用就是使该类只能创建一个对象。

首先将构造函数私有化，使其不能使用 new 关键字无限创建对象。

然后提供一个返回对象的静态方法，要想创建对象就使用该静态方法。

其次提供一个该对象类型的字段，默认为 null ，在静态方法中判断，如果该字段为空，就 new 一个对象给该字段，然后返回。如果不为空，那么代表该对象已经被创建，就直接返回。

这样自始至终都是使用的一个对象。

静态与非静态

静态构造函数不允许有参数，也不能有返回值和修饰符。

在一个类中，如果既有静态成员又有实例成员，则首先会初始化静态成员，并且只初始化一次，然后再初始化实例成员。

不管有多少实例，静态成员都只初始化一次，示例成员有多少示例就初始化多少次。

在整个项目中，静态成员是资源共享的。

静态成员只在整个项目结束的时候才释放资源。

静态成员存储在内存中的静态存储区域中。

XML

XML 是一种可拓展标记语言。

XML 虽然与 HTML 长相相似，但是作用却不同， HTML 的作用是显示网页的内容， XML 的作用是存储数据。另外 XML 区分大小写。

XML 存储数据的方便之处就在于 XML 可读性强，数据存储规范。解析数据可以根据自定义的标签解析，方便快捷。

节点、元素、成对、有且只有一个根节点

XML 头部指定了 XML 文档的版本以及编码格式。

标签分为单闭合标签（类似 HTML 中 <br />）和双闭合标签（类似 HTML 中 <p></p>）

XML 文档有且只有一个根节点。根节点就是最外层的节点。

创建 XML

给节点赋值有两种方式

  • InnerText，会将值中的特殊字符转义，例如 <p>test</p> 会存储为 &lt;p&gt;test&lt;/p&gt;
  • InnerXml，不会转义特殊字符，会将类似 <p></p> 的数据当作节点来对待。

使用 price.SetAttribute("AttrName", "AttrValue"); 的方式可以给标签增加属性并赋值。
