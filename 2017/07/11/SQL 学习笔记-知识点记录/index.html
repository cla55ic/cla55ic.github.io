数据库类型

VARCHAR 和 NVARCHAR
NVARCHAR 比 VARCHAR 多占用一倍的空间，但是可以兼容更多类型的字符。
VARCHAR 和 CHAR
CHAR 是固定长度存储数据的，例如 CHAR(10) 存储字符串 abc，那么在 abc 会补七个空格，所以 CHAR 会比 VARCHAR 占用更多的空间，但是好处是索引效率高。

数据库迁移

  1. 分离、附加
  2. 脱机、附加
  3. 备份、还原
  4. 生成 SQL 脚本

约束

目的：保证数据完整性
种类：默认值约束、非空约束、主键约束、外键约束、唯一键约束、检查约束
约束并不是越多越好，增加约束好处是可以保证数据完整性和正确性，坏处是会牺牲一定的性能。

WHERE 条件过滤

多条件过滤： AND OR NOT (优先级：NOT > AND > OR)
区间查询：BETWEEN AND (双闭区间)

模糊查询

% ：匹配任意多个任意字符，包括 0 个
_ ：匹配一个任意字符
[]：范围，例如 区间：[0-9]，散列值[0,1,4,7]
特殊转义字符：用两个单引号表示数据库中的一个单引号。特殊字符使用[]括起来，例如 [[]

GROUP BY

分组查询选择列的范围：分组包含的列和聚合函数。
例如，SELECT order_type, count(1), sum(order_price),avg(order_price) FROM order GROUP BY order_type 。

HAVING

不同于 WHERE 是对选择列进行筛选，HAVING 子句是对分组后的组进行筛选。
例如，筛选出通话总时长大于 500 秒的呼叫员信息。首先对呼叫员编号列分组，然后使用 HAVING 进行筛选。
SELECT CALLERNUMBER, SUM(DATEDIFF(SECOND,STARTDATETIME,ENDDATETIME)) AS TOTAL_TIME FROM CALLRECORDS GROUP BY CALLERNUMBER HAVING SUM(DATEDIFF(SECOND,STARTDATETIME,ENDDATETIME)) > 500。

类型转换

CAST(要转换的表达式 AS 目标类型)CAST(NVARCHAR(32) AS ORDER_ID)

JOIN

JOIN 用于根据两个或多个表之间的关系，从这些表中查询数据。
因为各个表的关联性，有时我们需要获取到的信息分布再不同的表中，这些表又通过某些列进行关联，要一次取得我们需要的信息，就要使用 JOIN 关键字。
PERSONS 表中有 ID_P、NAME、ADDRESS 共三列，OERDERS 表中有 ORDER_NO 、ID_P 两列。两张表通过 ID_P 列联系了起来。
SELECT PERSONS.NAME, PERSONS.ADDRESS, ORDERS.ORDER_NO FROM PERSONS, ORDERS WHERE PERSONS.ID_P=ORDERS.ID_P 。
可以使用上述语句查出顾客和订单信息。也可使用 JOIN 来完成这些工作。
SELECT PERSONS.NAME, PERSONS.ADDRESS, ORDERS.ORDER_NO FROM PERSONS INNER JOIN ORDERS ON PERSONS.ID_P=ORDERS.ID_P 。
INNER JOIN 只是 JOIN 关键字的一种。INNER JOIN 为内连接，表示两表中至少存在一个匹配时才返回行。还有另外几种 JOIN 如下
下面列出几种可使用的 JOIN 类型和解释

JOIN 类型 解释
JOIN / INNER JOIN 两表中存在至少一个匹配才返回行
LEFT JOIN 返回左表所有的行，右表不匹配则列数据为空
RIGHT JOIN 返回右表中所有的行，左表不匹配则列数据为空
FULL JOIN 返回的行中包含左右表所有数据，不匹配的列数据为空

其他类型的 JOIN 关键字用法与 INNER JOIN 相似

UNION

UNION 关键字合并两个 SELECT 查询出的结果集。要求两个查询的列数相同，列类型相似，并且列的顺序一致。
UNION 会去除重复的数据，不论重复是存在于一个表中，还是两表之间，UNION 只会显示一条数据。
UNION 会显示所有的数据，即使数据存在重复。
另外，使用 UNION 查询出的结果集的列名总是与第一个 SELECT 中的列名一致。
UNION 使用的简单示例
SELECT NAME FROM TABLEA UNION SELECT NAME FROM TABLEB

  1. 查询通话时间最长的5条记录
    SELECT TOP 5 * FROM CALLRECORDS ORDER BY DATEDIFF(SECOND, START_DATETIME, END_DATETIME)
  2. 查询以0开头的通话总时,以秒为计算单位
    SELECT *, DATEDIFF(SECOND, START_DATETIME, END_DATETIME) FROM CALLRECORDS WHERE TEL_NUM LIKE '0%'
  3. 查询2010年7月通话总时长最多的前两个呼叫员的编号

  4. 查询2010年7月拨打电话次数最多的前两个呼叫员的编号
    SELECT TOP 2 CALLER_NUM, COUNT(*) AS CALL_COUNT FROM CALLRECORDS WHERE DATEDIFF(MONTH,START_DATETIME,'2017-7-1')=0 GROUP BY CALLER_NUM ORDER BY CALL_COUNT DESC
